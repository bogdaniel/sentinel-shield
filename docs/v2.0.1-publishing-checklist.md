## Publish gates (require separate explicit authorization)

Owner instruction alone is **not** sufficient. The release-authorization flow requires
both a governed authorization record (bound by a `--confirm-token`) **and** an explicit
destructive step (`--execute`); no tool creates a tag without it.

- [ ] Evidence PR merged to `master`
- [ ] Final tag target confirmed against merged evidence (metadata-only)
- [ ] Explicit release-owner instruction to publish received
- [ ] Governed authorization recorded:
      `authorize-production-release.sh authorize --candidate evidence/releases/v2.0.1-ga-candidate.json --source-commit <TAG_TARGET> --authorization <record> --confirm-token <token> --output evidence/releases/v2.0.1-authorization.json`
      The authorization record must additionally match the verified candidate fingerprint
      (`candidate_hash`) and every identity field (version, stage, scope, source_commit, tag),
      use **distinct** `requested_by` / `approved_by` (self-approval is refused), and carry a
      non-expired `expires_at` — otherwise `authorize` fails closed.
- [ ] Exact commands printed for the AUTHORIZED candidate:
      `authorize-production-release.sh print-tag-commands --candidate evidence/releases/v2.0.1-ga-candidate.json --authorization evidence/releases/v2.0.1-authorization.json`
- [ ] Signed tag created via the **destructive** flag:
      `finalize-release-evidence.sh --evidence evidence/releases/v2.0.1-ga.json --mode source-tag --tag v2.0.1 --execute`
      (or `git tag -s v2.0.1 <TAG_TARGET> -m "Sentinel Shield v2.0.1 — Engine-Only Maintenance Release"`)
- [ ] Tag pushed
- [ ] GitHub Release created from `docs/v2.0.1-release-notes.md`
