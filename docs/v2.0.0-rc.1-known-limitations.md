# Sentinel Shield v2.0.0-rc.1 — Known Limitations

Scope: **engine-only** release candidate. This document is the authoritative
limitations statement referenced by the release authorization.

## Not validated in this release

- **Laravel live-consumer validation** — not included. No Laravel profile has been
  exercised against a live consumer repository. `framework-validated` is not claimed.
- **Symfony live-consumer validation** — not included. Same as above.
- **Framework-validated RC** — not authorized.
- **Full-platform GA** — not authorized.

## Validated, with stated boundaries

- **Engine** — `engine-tested` via the engine's own default-branch CI and the local
  production-readiness suite.
- **PHP-library / Node / React consumers** — `standalone-consumer-tested` against
  controlled fixtures, not live production adoption.
- **Adopter harness** — `black-box-adopter-harness-tested`; controlled fixtures, not
  independent production adopters.

## Environment-bound evidence caveats

- **Process-group isolation (bounded-process):** on hosts without `perl`/job control and
  with `SENTINEL_SHIELD_BP_REQUIRE_ISOLATION=1`, the launcher fails closed (refuses to
  launch) — this is correct defensive behavior, verified by the bounded-process suites.
  macOS bash 3.2 can mask Linux-CI process-control behavior; CI (ubuntu) is authoritative.
- **Live full-scanner security acceptance:** the CI-built security summary carries no
  blocking findings and the acceptance engine is proven fail-closed. A single end-to-end
  `security-acceptance.json` over the complete production scanner manifest is produced by
  the authoritative ci-security run, not reconstructed offline.

## Soak

An RC soak period (recommended 7 calendar days) is required before publication is
recommended. See the release process runbook.
