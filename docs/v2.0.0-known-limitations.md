# Sentinel Shield v2.0.0 — Known Limitations

Scope: **engine-only** production release. This document is the authoritative
limitations statement referenced by the release authorization (GATE 10).

## Not validated in this release

- **Laravel live-consumer validation** — not included. No Laravel profile has been
  exercised against a live consumer repository. `framework-validated` is not claimed.
- **Symfony live-consumer validation** — not included. Same as above.
- **Framework-validated GA** — not authorized (BLOCKED by the release-authorization gate).
- **Full-platform GA** — not authorized (BLOCKED by the release-authorization gate).

## Validated, with stated boundaries

- **Engine** — `engine-tested` via the engine's own default-branch CI (7 required
  workflows green at the release source) and the local production-readiness suite.
- **PHP-library / Node / React consumers** — `standalone-consumer-tested` against
  controlled fixtures, not live production adoption.
- **Adopter harness** — `black-box-adopter-harness-tested`; controlled fixtures, not
  independent production adopters.

## Environment-bound evidence caveats

- **Process-group isolation (bounded-process):** on hosts without `perl`/job control and
  with `SENTINEL_SHIELD_BP_REQUIRE_ISOLATION=1`, the launcher fails closed (refuses to
  launch) — correct defensive behavior, verified by the bounded-process suites. macOS
  bash 3.2 can mask Linux-CI process-control behavior; CI (ubuntu) is authoritative.
- **Live full-scanner security acceptance:** the single end-to-end
  `security-acceptance.json` over the complete production scanner manifest is produced by
  the authoritative `ci-security` run (event=push, branch=master, head=13be630), not
  reconstructed offline. Decision `accepted`, 0 blocking findings, 0 violations.

## Soak — release-owner waiver (recorded)

The RC soak period (recommended 7 calendar days) is a **recommendation**, not a technical
gate. `v2.0.0-rc.1` was published on 2026-07-08 from source
`13be630a64d044e0ffca8850e3424c2f0799aebf`, starting the soak clock.

On **2026-07-09** the release owner **explicitly waived the remaining soak window** to
proceed directly to `v2.0.0` engine-only GA. Justification: the GA release source is the
**same executable commit** `13be630` from which rc.1 was cut — the diff from the rc.1
evidence merge is evidence/docs metadata only — so the equivalent beta/RC/production-
readiness evidence already exists and no executable change is being shipped past the RC.

This waiver is recorded here, in the evidence PR body, and encoded as
`evidence.soak_complete = true` in the production-readiness report with this rationale. No
technical gate was bypassed: every gate in `authorize-production-release.sh
verify-candidate` (source CI, artifacts, manifest, security acceptance, compatibility,
adopter, upgrade, rollback, limitations, support, incident-response) is independently
re-derived and green.
