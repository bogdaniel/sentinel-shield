## Publish gates (require separate explicit authorization)

Owner instruction alone is **not** sufficient. The release-authorization flow requires
both a governed authorization record (bound by a `--confirm-token`) **and** an explicit
destructive step (`--execute`); no tool creates a tag without it.

- [ ] Evidence PR merged to `master`
- [ ] Final tag target confirmed against merged evidence (`G1..G2` metadata-only)
- [ ] Explicit release-owner instruction to publish received
- [ ] Governed authorization recorded (candidate paths are descriptor-relative — no `--base-dir` needed):
      `authorize-production-release.sh authorize --candidate evidence/releases/v2.0.0-candidate.json --source-commit <TAG_TARGET> --authorization <record> --confirm-token <token> --output evidence/releases/v2.0.0-authorization.json`
- [ ] Exact commands printed for the AUTHORIZED candidate:
      `authorize-production-release.sh print-tag-commands --candidate evidence/releases/v2.0.0-candidate.json --authorization evidence/releases/v2.0.0-authorization.json`
- [ ] Signed tag created via the **destructive** flag:
      `finalize-release-evidence.sh --evidence evidence/releases/v2.0.0-ga.json --mode source-tag --tag v2.0.0 --execute`
      (or the printed `git tag -s v2.0.0 <TAG_TARGET> -m "Sentinel Shield v2.0.0 — Engine-Only Production Release"`)
- [ ] Tag pushed
- [ ] GitHub Release created from `docs/v2.0.0-release-notes.md`
